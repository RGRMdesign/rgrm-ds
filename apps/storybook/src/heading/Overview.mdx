# Heading

Heading typography for the RGRM design system. Visual styles live in
`@rgrmdesign/rgrm-ds-css`; React and the web component are thin wrappers that apply
the same BEM classes.

Load `@rgrmdesign/rgrm-ds-tokens` once in your app so the CSS custom properties are
defined.

**Design spec** — <a href={FIGMA_LIBRARY_URL}>View in Figma</a> for anatomy, variants,
and usage notes.

## Level and appearance

Headings separate **semantic level** from **visual appearance**:

- **Level** — the `<h1>`–`<h6>` element that is rendered. This drives the document
  outline and accessibility.
- **Appearance** — the typography scale applied via a BEM modifier (`.rgrm-heading--h1`
  … `.rgrm-heading--h6`, or `.rgrm-heading--display`).

When `appearance` is omitted, it defaults to `level`. You can combine them freely — for
example, render an `<h2>` with the visual scale of appearance 4.

## Choose an implementation

Pick a page in the sidebar under **Typography → Heading**, or jump directly:

- **React** — [`@rgrmdesign/rgrm-ds-react`](?path=/docs/typography-heading-react--documentation) for React apps
- **CSS** — [`@rgrmdesign/rgrm-ds-css`](?path=/docs/typography-heading-css--documentation) for hand-written HTML or any stack without a wrapper
- **Web component** — [`@rgrmdesign/rgrm-ds-elements`](?path=/docs/typography-heading-web-components--documentation) for vanilla JS, server-rendered HTML, or non-React frameworks

Each documentation page includes install steps, copy-paste examples, and all appearance
variants.
